Overview

Project Overview

AvionPure is an IT solutions company specialising in Security & Observability, Cloud Infrastructure, Software Development, and Technology Consulting. The website is a single-page corporate site with a linked job application page. It is built entirely with vanilla HTML, CSS, and JavaScript — no frameworks or build tools required.

🌐 Website Type

Corporate marketing site with career applications. Static HTML — no server-side code required. Can be hosted on any static host (Netlify, GitHub Pages, S3).

🎨 Design Theme

Light / white background with cyan (#0095bb) and purple (#6d28d9) brand accents. No dark mode.

📱 Responsive

Mobile-first responsive design. Breakpoints at 1024px, 860px, 768px, 600px, and 480px. Hamburger menu on mobile.

⚙️ Dependencies

Google Fonts (Inter + Space Grotesk). All other code is self-contained. No npm, no bundler, no external JS libraries.

apply.html — Job Application Page

A separate page that opens in a new tab when "Apply Now" is clicked on any job card. It reads the ?job= URL parameter and dynamically renders the matching job details and application form.

URL Parameters

URLJob Loaded
apply.html?job=sre-engineerSRE Engineer
apply.html?job=siem-architectSIEM Architect
apply.html?job=observability-engineerObservability Engineer
apply.html?job=security-engineerSecurity Engineer
apply.html?job=anything-else"Position Not Found" message

Layout — Two Columns

Left Panel — Job Detail (sticky)

Shows: badge (Open / Hiring Now), job title, meta tags (type, location, department), Overview paragraph, Responsibilities list (7 bullets), Requirements list (7 bullets), Skills tags. Sticks to top while user scrolls the form.

Right Panel — Application Form

Fields: First Name, Last Name, Email, Phone, LinkedIn URL, Years of Experience (dropdown), Cover Letter (textarea), Resume upload (PDF/DOC/DOCX, max 5 MB). Submit button → success message on valid submission.

Form Validation

Client-side validation checks: First Name, Email, and Resume file are required. On missing fields the submit button plays a shake animation. On success, the form hides and a confirmation message appears.

Functionality

JavaScript Behaviours — script.js

1 · Navbar Scroll

Listens to window scroll. Adds .scrolled class to #navbar when scrollY > 40. The .scrolled class applies white background, blur, and box-shadow.

2 · Hamburger Menu

Click on #hamburger toggles .open class on #mobileMenu. The .open class animates max-height from 0 to 300px. Any link click closes the menu.

3 · Particle Canvas

IIFE in script.js. Renders 80 particles on #particleCanvas using requestAnimationFrame. Particles drift slowly with random velocity. Particles within 120px of each other are connected with a semi-transparent cyan line. Resizes on window resize event.

4 · Scroll Reveal

IntersectionObserver (threshold 0.1, root margin -40px bottom) watches: .service-card, .job-card, .contact-card, .pillar, .tech-badge, .section-header, .about-text. Adds .reveal class initially (opacity 0, translateY 30px), then adds .visible when in viewport (opacity 1, translateY 0). Staggered delays via inline transitionDelay per index.

5 · Active Nav Highlight

Second IntersectionObserver (threshold 0.4) watches all section[id] elements. When a section enters viewport, its corresponding nav link turns cyan; all others reset.

6 · Counter Animation (inactive)

Originally animated .stat-num elements counting up from 0 to target. The hero stats section was removed from HTML, so this code is present in script.js but never triggers. The observer checks if (statsEl) before observing, so it fails gracefully.

7 · Contact Form (index.html)

Validates Name, Email, Message. On invalid: shake animation on submit button. On valid: disables button, changes text to "Sending…", waits 1.2s, hides button, shows #formSuccess banner. Form submission is simulated (no backend).
